# 設計ドキュメント

## 構成

このディレクトリには cub3D プロジェクトの設計情報が格納されています。

```
docs/design/
├── README.md           # このファイル
├── overview.md         # プロジェクト全体の概要
├── architecture.md     # システムアーキテクチャ
├── modules/            # モジュール別設計
│   └── <module>/
│       ├── README.md   # モジュール概要(入口)
│       └── design.md   # 詳細設計書(実装仕様)
└── data/               # データ構造定義
```

## ドキュメント規約

### README.md と design.md の使い分け

各モジュールディレクトリには2種類のドキュメントを配置します:

#### README.md (モジュール概要)
**対象読者**: 初めてそのモジュールを見る人、全体像を把握したい人

**記載内容**:
- モジュールが**何を**するのか(責務の概要)
- **誰が**使うのか(呼び出し元/依存関係)
- 主要な制約・ルール(簡潔に)
- 関連ドキュメントへのリンク

**分量**: 5分程度で読める長さ(1-2画面)

#### design.md (詳細設計書)
**対象読者**: 実装者、コードレビュアー、デバッグ担当者

**記載内容**:
- 入力/出力の**正確な**仕様(型、フォーマット、前提条件)
- アルゴリズムの**具体的な**フロー(擬似コード、状態遷移図)
- 内部データ構造(構造体定義、フラグ管理)
- ヘルパー関数の責務と実装方針
- エラーハンドリングの**網羅的な**一覧
- テストケース、エッジケース

**分量**: 実装に必要な情報をすべて含む(制限なし)

### 例: parse/config モジュール

```
modules/parse/config/
├── README.md    → 「6要素(テクスチャ×4 + RGB×2)を順不同でパース」
└── design.md    → 「フラグ配列による状態管理、`.xpm`検証、RGB範囲チェック詳細」
```
